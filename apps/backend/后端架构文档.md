# 校园通信系统 - 后端架构文档

## 1. 技术栈与架构概览

### 1.1 技术栈
| 层级 | 技术 |
|------|------|
| 框架 | NestJS 11 |
| 语言 | TypeScript 5.7 |
| 数据库 | PostgreSQL |
| ORM | Prisma 7.8 |
| 实时通信 | Socket.IO 4.8 |
| 认证 | Passport + JWT |
| 密码加密 | bcrypt |
| 配置管理 | @nestjs/config |
| 文件上传 | Multer |

### 1.2 架构模式
采用 **NestJS 模块化架构**，遵循分层设计：
1. **Controller 层** - 接收 HTTP/WebSocket 请求，路由分发
2. **Service 层** - 业务逻辑处理
3. **DTO 层** - 数据传输对象与验证
4. **Guard/Interceptor 层** - 认证守卫与响应拦截
5. **Module 层** - 模块组织与依赖注入
6. **Prisma 层** - 数据库访问与模型定义

## 2. 数据库模型设计

### 2.1 模型关系图
```
User (1) ───< (N) Message (SentMessages)
User (1) ───< (N) Message (ReceivedMessages)
User (1) ───< (N) FileInfo
User (1) ───< (N) RoomMember

Room (1) ───< (N) RoomMember
Room (1) ───< (N) Message

RoomMember (N) ───> User (1)
RoomMember (N) ───> Room (1)
```

### 2.2 模型定义 (`prisma/schema.prisma`)

| 模型 | 功能 | 关键字段 |
|------|------|---------|
| **User** | 用户 | id, username, email, displayName, password, avatar, status, lastSeenAt |
| **Message** | 消息 | id, content, type, fileUrl, fileName, fileSize, senderId, receiverId, roomId |
| **Room** | 聊天室 | id, name, avatar, isGroup |
| **RoomMember** | 房间成员 | userId, roomId, joinedAt |
| **FileInfo** | 文件信息 | originalName, fileName, mimeType, size, url, uploadedById |

### 2.3 关系说明
- **User ↔ Message**: 一对多（发送/接收）
- **User ↔ Room**: 多对多（通过 RoomMember 中间表）
- **Room ↔ Message**: 一对多
- **User ↔ FileInfo**: 一对多

| `typing:start` | 客户端→服务端 | 通知对方正在输入 |

### 4.2 各模块功能

#### PrismaModule / PrismaService
- **位置**: `libs/shared/src/prisma/`
- **功能**: 数据库连接管理
- **特性**:
  - 使用 `@prisma/adapter-pg` PostgreSQL 适配器
  - 从环境变量 `DATABASE_URL` 读取连接字符串
  - 继承 PrismaClient，可被注入到所有 Service 中
- **导出**: `PrismaService`

#### ResponseModule / ResponseService
- **位置**: `libs/shared/src/response/`
- **功能**: 统一 API 响应格式
- **接口**:
  ```typescript
  interface ApiResponse<T> {
    code: number;
    message: string;
    data: T;
    timestamp: string;
  }
  ```
- **方法**:
  - `success(data, message, code)` - 成功响应
  - `paginated(data, total, page, pageSize, message)` - 分页响应
  - `isStandardFormat(data)` - 判断是否已是标准格式
- **导出**: `ResponseService`

#### InterceptorModule / InterceptorService
- **位置**: `libs/shared/src/interceptor/`
- **功能**: 全局 HTTP 响应拦截器
- **职责**:
  1. 自动将 Controller 返回数据包装为标准 API 格式
  2. 记录请求日志（method, url, statusCode, duration）
  3. 避免对已是标准格式的响应二次包装
- **注册位置**: `main.ts` 全局注册 (`app.useGlobalInterceptors`)
- **导出**: `InterceptorService`

#### SharedModule
- **位置**: `libs/shared/src/shared.module.ts`
- **特性**: `@Global()` 全局模块
- **导出**: `SharedService`, `PrismaModule`, `InterceptorModule`, `ResponseModule`
- **作用**: 所有模块无需显式导入即可使用 PrismaService 和 ResponseService

## 5. 认证与安全机制

### 5.1 JWT 认证流程
```
客户端请求
  ↓
JwtAuthGuard (路由守卫)
  ↓
JwtStrategy (Passport策略)
  - 从 Authorization Header 提取 Bearer Token
  - 验证 Token 签名和过期时间
  - 查询数据库确认用户存在
  ↓
将用户信息注入 req.user
  ↓
Controller / Service 处理业务
```

### 5.2 WebSocket 认证
```
Socket.IO 连接
  ↓
ws-auth.middleware.ts
  - 从 handshake.auth.token 或 headers.authorization 提取 Token
  - JwtService.verify() 验证
  - 将用户信息写入 socket.data.user
  ↓
EventsGateway.handleConnection()
  - 读取 socket.data.user
  - 更新数据库在线状态
  - 广播上线通知
```

### 5.3 密码安全
- 使用 `bcrypt` 哈希密码（saltRounds = 10）
- 数据库不存储明文密码
- 登录时 `bcrypt.compare()` 验证

## 6. 数据流向

### 6.1 HTTP 请求处理流程
```
Client Request
  ↓
Global Prefix (/api)
  ↓
JwtAuthGuard (如需认证)
  ↓
Controller (路由分发)
  ↓
Service (业务逻辑)
  ↓
PrismaService (数据库操作)
  ↓
ResponseService (格式化响应)
  ↓
InterceptorService (日志记录 + 自动包装)
  ↓
Client Response
```

### 6.2 实时消息流程
```
Client A 发送消息
  ↓
Socket.IO emit('sendMessage')
  ↓
EventsGateway.handleSendMessage()
  - JWT 认证（中间件）
  - ChatService.createMessage() 保存到数据库
  - 如果是房间消息：server.to(`room:${roomId}`).emit('message')
  - 如果是私聊：server.to(`user:${receiverId}`).emit('message')
  ↓
Client B 接收消息
  Socket.IO on('message')
```

### 6.3 在线状态同步流程
```
用户连接 WebSocket
  ↓
handleConnection()
  - 首次连接：更新 status='online'
  - server.emit('userOnline', {...})
  ↓
用户断开 WebSocket
  ↓
handleDisconnect()
  - 所有连接断开：更新 status='offline'
  - server.emit('userOffline', {...})
```

## 10. 总结

该后端采用 **NestJS 模块化架构**，实现了：
- **清晰的模块边界**: 8个业务模块各司其职，依赖关系明确
- **统一的响应格式**: 通过 ResponseService + InterceptorService 实现全局标准响应
- **完整的认证体系**: JWT HTTP认证 + WebSocket认证双模式
- **实时通信能力**: Socket.IO 实现消息推送、在线状态同步
- **数据库抽象**: Prisma ORM + PostgreSQL 适配器，类型安全
- **全局共享能力**: @Global() SharedModule 提供 PrismaService 和 ResponseService
- **自动化数据初始化**: SeedModule 应用启动时自动创建测试数据

整体架构遵循 NestJS 最佳实践，依赖注入解耦，易于测试和扩展。
