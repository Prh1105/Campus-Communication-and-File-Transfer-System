# 校园通信系统 - 前端架构文档

## 1. 整体框架设计

### 1.1 技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **路由**: React Router v6
- **状态管理**: React Context API
- **实时通信**: Socket.IO Client
- **HTTP 客户端**: Axios
- **样式**: Tailwind CSS
- **类型定义**: @campus-im/shared (共享类型包)

### 1.2 架构模式
采用**分层架构**设计，从上到下分为：
1. **路由层** - 页面路由配置
2. **Context 层** - 全局状态管理
3. **页面层** - 页面级组件
4. **组件层** - 可复用 UI 组件
5. **服务层** - API 调用封装
6. **工具层** - 基础配置和工具

## 3. Context 层级设计

### 3.1 Context 嵌套顺序
```
AuthProvider (最外层)
  ↓
SocketProvider (依赖 AuthProvider)
  ↓
ChatProvider (依赖 SocketProvider)
```

### 3.2 Context 详细说明

#### AuthContext (`src/contexts/AuthContext.tsx`)
**职责**: 用户认证状态管理
- **状态**:
  - `user`: 当前用户信息
  - `token`: JWT 认证令牌
  - `isAuthenticated`: 是否已认证
  - `isLoading`: 加载状态
- **方法**:
  - `login()`: 用户登录
  - `register()`: 用户注册
  - `logout()`: 用户登出
- **特性**:
  - Token 持久化到 localStorage
  - 初始化时自动验证 token
  - 监听 401 错误自动登出

#### SocketContext (`src/contexts/SocketContext.tsx`)
**职责**: WebSocket 连接管理
- **状态**:
  - `socket`: Socket.IO 实例
- **特性**:
  - 依赖 AuthProvider 的认证状态
  - 认证成功后自动连接
  - 连接状态日志记录
  - 组件卸载时自动断开

#### ChatContext (`src/contexts/ChatContext.tsx`)
**职责**: 聊天业务状态管理
- **状态**:
  - `messages`: 当前聊天消息列表
  - `onlineUsers`: 在线用户列表
  - `rooms`: 聊天室列表
  - `activeChat`: 当前激活的聊天目标
  - `typingUsers`: 正在输入的用户列表
- **方法**:
  - `sendMessage()`: 发送消息
  - `loadHistory()`: 加载聊天记录
  - `selectChat()`: 选择聊天目标
  - `setOnlineUsers()`: 设置在线用户
  - `setRooms()`: 设置聊天室
- **特性**:
  - 依赖 SocketContext 获取 socket 实例
  - 实时接收新消息
  - 支持私聊和群聊
  - 输入状态同步

## 4. 组件层设计

- **依赖**: `ChatContext`

### 4.2 列表组件

#### UserList (`src/components/UserList.tsx`)

- **子组件**: `UserCard`

- **Props**: `room`, `isActive`, `onClick`

#### ChatHeader (`src/components/ChatHeader.tsx`)
- **功能**: 聊天头部信息
- **依赖**: `ChatContext`
- **特性**:
  - 显示聊天对象名称
  - 显示在线状态或成员数
  - 根据聊天类型动态显示

#### MessageList (`src/components/MessageList.tsx`)
- **功能**: 消息列表展示
- **依赖**: `ChatContext`, `AuthContext`
- **子组件**: `MessageBubble`
- **特性**:
  - 自动滚动到底部
  - 空状态提示
  - 区分自己和他人的消息

#### MessageBubble (`src/components/MessageBubble.tsx`)
- **功能**: 单条消息气泡
- **Props**: `message`, `isMine`
- **特性**:
  - 支持文本、图片、文件、系统消息
  - 区分发送者样式
  - 文件可下载
  - 图片可预览

#### MessageInput (`src/components/MessageInput.tsx`)
- **功能**: 消息输入框
- **依赖**: `ChatContext`, `file.service`
- **特性**:
  - 文本输入
  - 文件上传
  - 上传进度显示

#### TypingIndicator (`src/components/TypingIndicator.tsx`)
- **功能**: 输入状态指示器
- **依赖**: `ChatContext`
- **特性**:
  - 显示正在输入的用户
  - 动画效果

### 4.5 工具组件

#### ProtectedRoute (`src/components/ProtectedRoute.tsx`)
- **功能**: 路由保护组件
- **依赖**: `AuthContext`
- **特性**:

#### ErrorBoundary (`src/components/ErrorBoundary.tsx`)
- **功能**: 错误边界组件

## 5. 服务层设计

### 5.1 API 配置 (`src/services/api.ts`)
- **职责**: Axios 实例配置和拦截器
- **特性**:
  - 请求拦截器：自动附加 JWT token
  - 响应拦截器：统一解包 `{code, data}` 格式
  - 错误处理：401 自动登出

### 5.2 服务模块

#### auth.service.ts
- `login()`: 用户登录
- `register()`: 用户注册
- `getMe()`: 获取当前用户信息

#### chat.service.ts
- `getPrivateHistory()`: 获取私聊记录
- `getRoomHistory()`: 获取群聊记录

#### room.service.ts
- `getRooms()`: 获取聊天室列表

#### user.service.ts
- `getAllUsers()`: 获取所有用户

#### file.service.ts
- `uploadFile()`: 文件上传

#### socket.ts
- `connectSocket()`: 建立 WebSocket 连接
- `disconnectSocket()`: 断开连接

## 6. 数据流向

### 6.1 认证流程
```
LoginPage/RegisterPage
  ↓
AuthContext.login()/register()
  ↓
auth.service API 调用
  ↓
保存 token 到 localStorage
  ↓
更新 user 状态
  ↓
SocketProvider 检测到认证状态
  ↓
建立 WebSocket 连接
```

### 6.2 聊天流程
```
用户选择聊天目标
  ↓
ChatContext.selectChat()
  ↓
ChatContext.loadHistory()
  ↓
chat.service API 调用
  ↓
更新 messages 状态
  ↓
MessageList 渲染消息
  ↓
用户发送消息
  ↓
ChatContext.sendMessage()
  ↓
Socket.emit('sendMessage')
  ↓
服务器广播消息
  ↓
Socket.on('message')
  ↓
更新 messages 状态
```

### 6.3 实时状态同步
```
Socket.IO 事件监听
  ↓
'users:update' → 更新在线用户
  ↓
'typing:start' → 显示输入状态
  ↓
'typing:stop' → 隐藏输入状态
  ↓
'message' → 接收新消息
```

## 8. 关键特性

### 8.1 类型安全
- 使用 TypeScript 严格模式
- 共享类型包 `@campus-im/shared`
- 完整的类型定义和推断

### 8.2 性能优化
- React.memo 避免不必要的重渲染
- useCallback 缓存函数引用
- useRef 避免闭包陷阱
- 虚拟滚动（消息列表）

### 8.3 用户体验
- 加载状态提示
- 错误边界保护
- 自动滚动到最新消息
- 输入状态实时显示
- 文件上传进度

### 8.4 安全性
- JWT 认证
- 路由保护
- Token 自动刷新
- 401 自动登出
